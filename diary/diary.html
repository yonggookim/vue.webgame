  • 대표님께 오후 교육을 받음 - 네트워큰은 기본적으로 실제 유선으로 연결되어 있다. (물론 그 외 라디오처럼 파형을 그리는 전파 방식도 있지만) - 미국은 정부자금이 직접 IT회사에 투자를 많이 한다. (오라클, 마이크로, ...) 그들의 배당금이 일정부분 다시 정부자금으로~ - MIS(Management Information System) : 경영 정보 시스템 기업을 시스템의 관점에서 봤을 때 이익창출을 위해 여러 하위 시스템들이 효율적으로 작동되도록 지원하는 시스템 결국.. MIS가 피터 드러커의 knowledge management(지식경영)이네.. - MRP(Material Requirements Planning) : 자재소요계획 최종제품이 필요한 시점으로부터 해당 제품의 제조 또는 조립에 필요한 하위 부품들이 필요되어지는 시점을 역으로 계산하는 시스템 또는 방법을 말한다. - SCM(Supply Chain Management) : 공급망 관리 부품 제공업자로부터 생산자 배포자. 고객에 이르는 물류의 흐름을 하나의 가치사슬 관점에서 파악하고 필요한 정보가 원활히 흐르도록 지원하는 시스템 - ERP(Enterprise Resource Planning) : 전산적 자원 관리 재무, 제조, 소매유통, 공급망, 인사관리, 운영전반의 비지니스 프로세스를 자동화하고 관리하는 시스템. - EIS(Executive Information Systems) : 중역정보시스템 DSS의 일종으로 최고 경영진에게 전략적인 의사 결정에 필요한 정보를 제공하는 체계 EP(Enterprise Portal) -> E-Commerce(카카오, NHN) -> 빅데이터 -> OTT(넷플렉스) - Microsoft RPC (원격 프로시저 호출) : 분산 된 클라이언트/서버 프로그램을 만들기 위한 강력한 기술을 정의 합니다. RPC 런타임 스텁 및 라이브러리는 네트워크 프로토콜과 통신에 관련 된 대부분의 프로세스를 관리 합니다. 이렇게 하면 네트워크 세부 정보가 아닌 응용 프로그램의 세부 정보에 집중할 수 있습니다. - IPC(Inter Process Communication) : 프로세스 간 통신 프로세스들 사이에 서로 데이터를 주고 받는 행위 또는 그에 대한 방법이나 경로를 뜻한다. - 클러스터링 기술 클러스터는 여러개의 시스템이 하나의 거대한 시스템으로 보이게 만드는 기술이다. 이렇게 하는 기술에는 여러가지가 있고 각 기술 특징을 이해하는 것이 필요하다. 이러한 클러스터는 디스크 공간 같은 하드웨어 자원을 공유할 수 있었고, 여러 사용자에게 컴퓨팅 자원을 제공할 수 있었다. 음.. IPC를 활용하여 클러스터링을 하는건가?
